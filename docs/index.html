Android SDK & Android Studio {How to setup for working environment?}

Working Environment

You need to setup your working environment for Android Development.

1. Install JDK

Need to install Java JDK from here

2. Install Android Studio

Need to install Android Studio as official IDE for Android Development.

3. Install Necessary SDK Package

You can install necessary SDK packages from Android Studio Menu -> Tools -> Android -> SDK Manager

Import Project Into Android Studio {How to import project?}

Import Project

You have to import the project into your android studio and copy into your workspace folder.

1. Where is project?

After you download from Codecanyon, there is the project source code at "Mobile Source\Mokets".

2. Import into Android Studio?

You could import from Android Studio Menu ( File -> New -> Import Project ). When popup window come out, please go to the folder(Mentioned at step 1). Yes, please copy that project into your work space.

3. Make sure the path?

You have to make sure for SDK and JDK path for your imported project. Right Click project -> Open Module Settings. At popup window you could check whether correct path or not.

Sync For Dependencies Libraries {How to download Dependencies Libraries?}

Dependencies Libraries

Dependencies Libraries are some additional libraries for the project. Normally, it declares at gradle file then Android Studio will sync for those libs into your local working environment.

1. Where is declared for Dependencies?

There is file so called build.gradle at "Mokets\app\build.gradle".

2. Sync Dependencies?

When Android Studio ask whether want to sync for those dependencies, please click to "sync" and wait until it finished successfully. Please make sure one thing to uncheck for "Offline work", if not it will sync and download from internet. (Android Studio -> Preferences -> Build, Execution, Deployment -> Build Tools -> Gradle )

Rename Package {How to rename the package?}

Rename Package

Currently, my original package name is "com.panaceasoft.mokets" so after you bought, you might need to change according your own package name. For example, if you want to change com.example.app to my.awesome.game. (Note : My example is using Android Studio)

1. Project pane

In your Project pane, click on the little gear icon

2. Compact Empty Middle Packages

Uncheck / De-select the Compact Empty Middle Packages option

3. Individual Directories

Your package directory will now be broken up in individual directories.

4. Individually select each directory you want to rename

- Right-click it
- Select Refactor
- Click on Rename
- In the Pop-up dialog, click on Rename Package instead of Rename Directory
- Enter the new name and hit Refactor
- Allow a minute to let Android Studio update all changes
- Note: When renaming com in Android Studio, it might give a warning. In such case, select Rename All

5. Gradle Build File

Now open your Gradle Build File (build.gradle - Usually app or mobile). Update the applicationId to your new Package Name and Sync Gradle, if it hasn't already been updated automatically:

6. activity_detail.xml

Please open activity_detail.xml from "res\layout\" then take a look this line app:layout_behavior="com.panaceasoft.mokets.utilities.ScrollingFABBehavior"
You might need to update your new package name at there also.

Google Map Key {How to generate api keys for Google Map?}

Google Map Key

When google change to Google Map V2, you might need to generate API access from Google Developer Console.

1. Login Google Developers Console

You might need to login into Google Developers Console using with your gmail account.

2. Create New Project

After login into Google Developer Console, please crate new project

3. Enable Google Map

After create project, please select that project first then "APIs & auth" -> "APIs". Enable "Google Maps Android API v2"

4. Get SHA1 Fingerprint

Run this command at terminal to get SHA1 Fingerprint
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
After that at your terminal window, you would see MD5, SHA1 and SHA256. Please copy SHA1 only.

5. Create New Key

You need to create the key from "APIs & auth" -> "Credentials" -> "Create new Key"

6. Android Key

You need to create "Android Key".

7. SHA1 Entry

You need to provide your SHA1 key which you already got since Step 4. After SHA1 key, you need to provide your app package name.

8. Take API Key

Please copy API to put at your android application.

9. Put Map API Key

Please put your Map Key API at AndroidManifext.xml.

Users Role & Access {About different user role and access right?}

There are Three types of user roles. They are

  • Administrator
    Full access for system such as add, edit, delete, publish and ban.
  • Manager
    Partial access for system such as add, edit, delete and publish.
  • User
    Basic access for system such as add and edit.

Default Users Account Information
Username : admin, Password : admin (Mokets Admin Account who can do everything on system.)
Username : manager, Password : manager (Manager Account with some limited access right.)

There are 4 shops with own shop admin account at my sample data. That shop admin account can do everything on their specific shop.
(For Shop 1) Username : shop1_admin, Password : shop1
(For Shop 2) Username : shop2_admin, Password : shop2
(For Shop 3) Username : shop3_admin, Password : shop3
(For Shop 4) Username : shop4_admin, Password : shop4
